Week 6 Technical Post - Class vs Instance Methods

Class and Instance methods are best summed up by stating that class methods that are called on classes, and instance methods are called on instances of the classes.


	class Sample
		def self.hello
    		puts 'hello'
  		end
  
  		def hilo
    		puts 'hilo'
  		end
	end

	Sample.hello # => "hello"
	Sample.hilo # => NoMethodError: undefined method ‘hilo’ for Sample:Class

	sampleinst = Sample.new
	sampleinst.hilo # => hilo
	sample.hello # => NoMethodError: undefined method ‘hello’ for #

As you can see, the class methods are defined as putting the "self" in front of the class method declaration. So that when you call the class method, you just type the Class name, followed by the method, as seen above with the hello method. If you had to call the hilo method on the class name, there would be an error with an undefined method.

With the instance method, when you create an instance of the class, as shown with "sampleinst", you use that name from thereon to call the instance methods, as shown with the hilo method above. If you tried to call the hello method on the instance sampleinst, then it would be a undefined method.

Basically, any time you are dealing with just the class, you would call the class method, and any time you were dealing with a specific created instance of a variable, you would call the insstance method. One example of something that could be used for a class method, is, for example, every time a new instance is created, a class variable is incremented by one. So you could create a class method to output the number of times an instance has been created, that's not accessible from instance methods.
